## 1. Executive Summary

### Purpose

PlanAlign Studio is the primary user interface for the PlanWise Design Matrix platform, enabling retirement plan consultants, account executives, and relationship managers to:

- View and edit plan design data with inline editing capabilities
- Perform regional peer benchmarking with statistical rigor
- Generate strategic assessments and gap analysis
- Create compliant client deliverables (PowerPoint presentations)
- Manage the 850+ client portfolio efficiently

### Design Philosophy

| Principle | Description |
|-----------|-------------|
| **Efficiency First** | Inline editing, keyboard navigation, and auto-save minimize clicks |
| **Progressive Disclosure** | Simple views for AEs, powerful tools for consultants |
| **Statistical Rigor** | All comparisons show confidence intervals, p-values, and cohort sizes |
| **Compliance Built-In** | Audit trails, k-anonymity, and pre-approved templates |
| **Responsive Design** | Optimized for laptop (primary) and tablet (secondary) use |

## 2. UI/UX Requirements

### 2.1 Functional Requirements

#### FR-001: Plan Design Matrix View
- Display all plan design fields organized by category
- Support inline click-to-edit functionality for all editable fields
- Show confidence scores for AI-extracted fields
- Display last updated timestamp and user for each field
- Provide quick link to field audit history

#### FR-002: Client Management
- Searchable, sortable client roster supporting 850+ clients
- Client creation workflow with required field validation
- Bulk operations (multi-select, bulk edit, bulk export)
- Risk indicators (data staleness, gaps, competitiveness)

#### FR-003: Regional Benchmarking
- Dynamic peer cohort construction (industry + geography + size)
- K-anonymity enforcement (minimum n=20 peers)
- Percentile positioning visualization (P25, P50, P75)
- Statistical significance indicators (p-values, effect sizes)
- Cohort transparency (show n-value and composition)

#### FR-004: Strategic Assessment
- Gap analysis against peer medians
- Plan competitiveness scoring (0-100)
- Data quality indicators (% verified fields)
- Recommendation generation with evidence strength
- Impact modeling (participation lift, cost analysis)

#### FR-005: Deliverable Generation
- One-click PowerPoint generation from pre-approved templates
- Export preview before download
- Compliance watermarking and disclaimer insertion
- Export logging for audit trail

### 2.2 Non-Functional Requirements

#### NFR-001: Performance
| Metric | Target |
|--------|--------|
| Initial page load | < 2 seconds |
| Client search response | < 500ms |
| Field save round-trip | < 1 second |
| Peer query (p95) | < 1.5 seconds |
| Peer query (p99) | < 3.0 seconds |
| PowerPoint generation | < 8 minutes (p95) |

#### NFR-002: Usability
- Maximum 3 clicks to reach any primary function
- Keyboard-only navigation support (Tab, Enter, Escape)
- Consistent visual language across all views
- Clear loading states and error messaging
- Undo capability for accidental edits (within session)

#### NFR-003: Accessibility
- WCAG 2.1 AA compliance
- Screen reader support (ARIA labels)
- Minimum contrast ratio 4.5:1
- Focus indicators on all interactive elements
- Support for reduced motion preference

### 2.3 Visual Design Requirements

#### Color Palette

```
Primary:
  - Fidelity Green: #008A4C (buttons, links, highlights)
  - Dark Green: #006838 (hover states)

Secondary:
  - Navy: #1A3668 (headers, navigation)
  - Slate: #4A5568 (body text)

Status Colors:
  - Success: #059669 (saved, complete)
  - Warning: #D97706 (needs review, low confidence)
  - Error: #DC2626 (validation errors)
  - Info: #2563EB (informational messages)

Neutrals:
  - Background: #F7FAFC
  - Card Background: #FFFFFF
  - Border: #E2E8F0
  - Disabled: #A0AEC0
```

#### Typography

```
Font Family: Inter (primary), system-ui (fallback)

Headings:
  - H1: 24px, 700 weight, Navy
  - H2: 20px, 600 weight, Navy
  - H3: 16px, 600 weight, Slate

Body:
  - Regular: 14px, 400 weight, Slate
  - Small: 12px, 400 weight, Slate
  - Caption: 11px, 400 weight, gray-500

Data:
  - Values: 14px, 500 weight, mono font (numbers)
  - Labels: 12px, 500 weight, uppercase tracking
```

#### Spacing System

```
Base unit: 4px

Spacing scale:
  - xs: 4px   (compact elements)
  - sm: 8px   (form elements)
  - md: 16px  (card padding)
  - lg: 24px  (section spacing)
  - xl: 32px  (page margins)
  - 2xl: 48px (major sections)
```

|-------|-----------|-----------|

| **Charts** | Recharts | React-native charts, accessible, customizable |

| **Tables** | TanStack Table | Headless, virtualization support, sorting/filtering |

### 6.4 Inline Editing Implementation

```typescript
// hooks/useInlineEdit.ts

import { useState, useCallback, useRef, useEffect } from 'react';
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { updateField } from '@/api/clients';
import { useDebounce } from './useDebounce';

interface UseInlineEditOptions {
  clientId: string;
  fieldName: string;
  initialValue: string | number | boolean;
  onSuccess?: () => void;
  onError?: (error: Error) => void;
  debounceMs?: number;
  requireReason?: boolean;
}

interface EditState {
  isEditing: boolean;
  isSaving: boolean;
  error: string | null;
  showSuccess: boolean;
}

export function useInlineEdit({
  clientId,
  fieldName,
  initialValue,
  onSuccess,
  onError,
  debounceMs = 500,
  requireReason = false,
}: UseInlineEditOptions) {
  const [value, setValue] = useState(initialValue);
  const [editState, setEditState] = useState<EditState>({
    isEditing: false,
    isSaving: false,
    error: null,
    showSuccess: false,
  });

  const queryClient = useQueryClient();
  const inputRef = useRef<HTMLInputElement>(null);
  const originalValue = useRef(initialValue);

  // Debounced value for auto-save
  const debouncedValue = useDebounce(value, debounceMs);

  const mutation = useMutation({
    mutationFn: (params: { value: any; reason?: string; notes?: string }) =>
      updateField(clientId, fieldName, params.value, params.reason, params.notes),
    onMutate: async (params) => {
      // Optimistic update
      await queryClient.cancelQueries({ queryKey: ['planData', clientId] });
      const previous = queryClient.getQueryData(['planData', clientId]);
      queryClient.setQueryData(['planData', clientId], (old: any) => ({
        ...old,
        [fieldName]: params.value,
      }));
      return { previous };
    },
    onSuccess: () => {
      originalValue.current = value;
      setEditState((s) => ({ ...s, isSaving: false, showSuccess: true }));
      setTimeout(() => setEditState((s) => ({ ...s, showSuccess: false })), 2000);
      onSuccess?.();
    },
    onError: (error: Error, _, context) => {
      // Rollback on error
      queryClient.setQueryData(['planData', clientId], context?.previous);
      setValue(originalValue.current);
      setEditState((s) => ({ ...s, isSaving: false, error: error.message }));
      onError?.(error);
    },
  });

  const startEdit = useCallback(() => {
    setEditState((s) => ({ ...s, isEditing: true, error: null }));
    setTimeout(() => inputRef.current?.focus(), 0);
  }, []);

  const cancelEdit = useCallback(() => {
    setValue(originalValue.current);
    setEditState((s) => ({ ...s, isEditing: false, error: null }));
  }, []);

  const saveEdit = useCallback(
    (reason?: string, notes?: string) => {
      if (value === originalValue.current) {
        setEditState((s) => ({ ...s, isEditing: false }));
        return;
      }
      setEditState((s) => ({ ...s, isSaving: true }));
      mutation.mutate({ value, reason, notes });
    },
    [value, mutation]
  );

  // Keyboard handlers
  const handleKeyDown = useCallback(
    (e: React.KeyboardEvent) => {
      if (e.key === 'Enter' && !requireReason) {
        e.preventDefault();
        saveEdit();
        setEditState((s) => ({ ...s, isEditing: false }));
      } else if (e.key === 'Escape') {
        cancelEdit();
      } else if (e.key === 'Tab') {
        saveEdit();
        // Let default tab behavior proceed
      }
    },
    [saveEdit, cancelEdit, requireReason]
  );

  return {
    value,
    setValue,
    editState,
    inputRef,
    startEdit,
    cancelEdit,
    saveEdit,
    handleKeyDown,
  };
}
```

### 6.5 Component Example: InlineEdit

```tsx
// components/common/InlineEdit.tsx

import React from 'react';
import { cn } from '@/utils/cn';
import { Check, X, Loader2, AlertCircle } from 'lucide-react';
import { useInlineEdit } from '@/hooks/useInlineEdit';

interface InlineEditProps {
  clientId: string;
  fieldName: string;
  initialValue: string | number;
  type?: 'text' | 'number' | 'percentage' | 'currency';
  suffix?: string;
  className?: string;
  requireReason?: boolean;
}

export function InlineEdit({
  clientId,
  fieldName,
  initialValue,
  type = 'text',
  suffix,
  className,
  requireReason = false,
}: InlineEditProps) {
  const {
    value,
    setValue,
    editState,
    inputRef,
    startEdit,
    cancelEdit,
    saveEdit,
    handleKeyDown,
  } = useInlineEdit({
    clientId,
    fieldName,
    initialValue,
    requireReason,
  });

  const formatValue = (val: string | number) => {
    if (type === 'percentage') return `${val}%`;
    if (type === 'currency') return `$${Number(val).toLocaleString()}`;
    return String(val);
  };

  // Read-only display
  if (!editState.isEditing) {
    return (
      <button
        onClick={startEdit}
        className={cn(
          'group inline-flex items-center gap-2 px-3 py-1.5 rounded-md',
          'text-slate-700 bg-slate-50 hover:bg-slate-100',
          'border border-transparent hover:border-slate-200',
          'transition-colors cursor-pointer',
          editState.showSuccess && 'bg-green-50 border-green-200',
          className
        )}
      >
        <span className="font-medium">
          {formatValue(value)}
          {suffix && <span className="text-slate-500 ml-1">{suffix}</span>}
        </span>
        {editState.showSuccess && (
          <Check className="w-4 h-4 text-green-600" />
        )}
      </button>
    );
  }

  // Edit mode
  return (
    <div className="relative inline-flex items-center gap-2">
      <input
        ref={inputRef}
        type={type === 'number' || type === 'percentage' || type === 'currency' ? 'number' : 'text'}
        value={value}
        onChange={(e) => setValue(e.target.value)}
        onKeyDown={handleKeyDown}
        onBlur={() => !requireReason && saveEdit()}
        step={type === 'percentage' ? '0.1' : '1'}
        className={cn(
          'w-24 px-3 py-1.5 rounded-md',
          'border-2 border-blue-500 focus:border-blue-600',
          'outline-none',
          editState.error && 'border-red-500'
        )}
        disabled={editState.isSaving}
      />

      {suffix && <span className="text-slate-500">{suffix}</span>}

      {editState.isSaving && (
        <Loader2 className="w-4 h-4 text-blue-500 animate-spin" />
      )}

      {editState.error && (
        <div className="absolute top-full mt-1 left-0 flex items-center gap-1 text-red-600 text-sm">
          <AlertCircle className="w-4 h-4" />
          {editState.error}
        </div>
      )}

      <button
        onClick={cancelEdit}
        className="p-1 rounded hover:bg-slate-100"
        aria-label="Cancel"
      >
        <X className="w-4 h-4 text-slate-500" />
      </button>
    </div>
  );
}
```

### 8.2 API Client Configuration

```typescript
// api/client.ts

import axios from 'axios';
import { QueryClient } from '@tanstack/react-query';

export const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000/api/v1',
  headers: {
    'Content-Type': 'application/json',
  },
});

// Request interceptor for auth
apiClient.interceptors.request.use((config) => {
  const token = localStorage.getItem('auth_token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// Response interceptor for error handling
apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // Handle auth error
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

// Query client configuration
export const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 5 * 60 * 1000, // 5 minutes
      gcTime: 30 * 60 * 1000, // 30 minutes (formerly cacheTime)
      retry: 3,
      refetchOnWindowFocus: true,
    },
    mutations: {
      retry: 1,
    },
  },
});
```

### 8.3 API Hook Examples

```typescript
// api/clients.ts

import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import { apiClient } from './client';
import type { Client, PlanData, FieldUpdate } from '@/types';

// Fetch all clients
export function useClients(filters?: {
  industry?: string;
  region?: string;
  search?: string;
}) {
  return useQuery({
    queryKey: ['clients', filters],
    queryFn: async () => {
      const { data } = await apiClient.get<Client[]>('/clients', {
        params: filters,
      });
      return data;
    },
  });
}

// Fetch single client plan data
export function usePlanData(clientId: string | null) {
  return useQuery({
    queryKey: ['planData', clientId],
    queryFn: async () => {
      if (!clientId) return null;
      const { data } = await apiClient.get<PlanData>(`/clients/${clientId}`);
      return data;
    },
    enabled: !!clientId,
  });
}

// Update field mutation
export function useUpdateField() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: async ({
      clientId,
      fieldName,
      value,
      reason,
      notes,
    }: FieldUpdate) => {
      const { data } = await apiClient.patch(
        `/clients/${clientId}/fields/${fieldName}`,
        { value, reason, notes }
      );
      return data;
    },
    onSuccess: (_, variables) => {
      // Invalidate and refetch
      queryClient.invalidateQueries({
        queryKey: ['planData', variables.clientId],
      });
    },
  });
}

// Fetch peer benchmarks
export function useBenchmarks(
  clientId: string | null,
  cohortConfig: { industry?: string; region?: string; size?: string }
) {
  return useQuery({
    queryKey: ['benchmarks', clientId, cohortConfig],
    queryFn: async () => {
      if (!clientId) return null;
      const { data } = await apiClient.get(`/clients/${clientId}/peers`, {
        params: cohortConfig,
      });
      return data;
    },
    enabled: !!clientId,
    staleTime: 60 * 60 * 1000, // 1 hour for benchmark data
  });
}
```

## 9. Performance Requirements

### 9.1 Performance Targets

| Metric | Target | Measurement |
|--------|--------|-------------|
| Time to Interactive (TTI) | < 3 seconds | Lighthouse |
| First Contentful Paint (FCP) | < 1.5 seconds | Lighthouse |
| Largest Contentful Paint (LCP) | < 2.5 seconds | Lighthouse |
| Cumulative Layout Shift (CLS) | < 0.1 | Lighthouse |
| Client list load (850 clients) | < 500ms | API response time |
| Field save round-trip | < 1 second | User-perceived |
| Peer query (p95) | < 1.5 seconds | Server metrics |

### 9.2 Performance Optimization Strategies

#### Client-Side Optimizations

1. **Code Splitting**
   ```typescript
   // Lazy load routes
   const Assessment = lazy(() => import('@/components/assessment/Assessment'));
   const Scorecard = lazy(() => import('@/components/scorecard/Scorecard'));
   ```

2. **Virtualized Lists**
   ```typescript
   // Use @tanstack/react-virtual for large lists
   import { useVirtualizer } from '@tanstack/react-virtual';

   const rowVirtualizer = useVirtualizer({
     count: clients.length,
     getScrollElement: () => parentRef.current,
     estimateSize: () => 48,
     overscan: 10,
   });
   ```

3. **Memoization**
   ```typescript
   // Memoize expensive calculations
   const sortedClients = useMemo(() => {
     return [...clients].sort((a, b) => a.name.localeCompare(b.name));
   }, [clients]);
   ```

4. **Image Optimization**
   - Use WebP format with fallbacks
   - Lazy load images below the fold
   - Use appropriate image sizes

#### Server-Side Optimizations

1. **Query Caching**
   - DuckDB materialized views for common aggregations
   - Redis cache for frequent queries
   - Pre-computed peer cohorts

2. **Database Indexes**
   - Index on industry, region, size for filtering
   - Composite index for common query patterns

3. **Response Compression**
   - GZIP compression for all API responses
   - JSON streaming for large datasets
